# 📌 Project Overview

The **School Library Management System (E-Library)** is a digital platform designed to modernize and automate traditional library operations within academic institutions. The system addresses inefficiencies in manual book tracking, delayed reporting, and limited access to book availability information.

This project supports **SDG 4: Quality Education** by improving access to educational resources, streamlining library services, and enhancing the learning experience for students and staff.

---

# 🎯 Problem Statement

Many school libraries still rely on manual or semi-manual processes for cataloging, borrowing, and monitoring books. These traditional methods lead to:

- Delayed book tracking
- Human errors in inventory
- Inefficient reporting
- Poor student accessibility
- Risk of data loss during system interruptions

The proposed E-Library system aims to solve these problems by implementing a centralized digital management system with real-time updates, automated reporting, and searchable catalog access.

---

# 🏗️ System Architecture

The project uses a **3-Tier Architecture** to ensure scalability, maintainability, and security.

### Architecture Layers
- **Presentation Layer** – User Interface for students, librarians, and administrators
- **Application Logic Layer** – Handles borrowing, returning, searching, and reporting processes
- **Data Layer** – Stores library records, user accounts, and transaction logs

This architecture allows easier maintenance and future scalability for additional features such as mobile integration and notification systems.

# 🧩 Core System Features

## Functional Requirements
- 📖 Online Public Access Catalog (OPAC)
- 🔄 Real-time book availability updates
- 🧾 Automated delinquency reporting
- 👥 Role-Based Access Control (RBAC)
- 📊 Monthly circulation reports
- 🗂️ Inventory management system

## Non-Functional Requirements
- ⚡ Fast search response time (< 2 seconds)
- 🔒 Secure access to student records
- 💾 Automated daily backups
- 🖥️ User-friendly interface
